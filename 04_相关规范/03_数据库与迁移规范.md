## 2. 表设计规范

### 2.1 命名

| 元素 | 规则 | 示例 |
| --- | --- | --- |
| 表名 | snake_case 复数 | `costs`, `conversation_messages` |
| 字段名 | snake_case | `farm_id`, `created_at` |
| 索引名 | `idx_<table>_<col>` 或 `ix_<table>_<col>` | `idx_costs_farm_id` |
| 外键名 | `fk_<from>_<to>` | `fk_costs_farm` |
| 唯一约束 | `uq_<table>_<col>` | `uq_users_username` |

### 2.2 通用字段

每张表必须有：

```python
id: int             # 自增主键
created_at: datetime  # 创建时间，默认 CURRENT_TIMESTAMP
updated_at: datetime  # 更新时间，ON UPDATE CURRENT_TIMESTAMP
farm_id: int           # 农场隔离（业务表必填）
```

软删除（按需）：

```python
deleted_at: datetime | None  # 软删除时间，None 表示未删除
```

### 2.3 类型选择

| Python | MySQL |
| --- | --- |
| `str` (短) | `VARCHAR(n)` |
| `str` (长) | `TEXT` |
| `str` (定长) | `CHAR(n)` |
| `int` | `INT` / `BIGINT` |
| `bool` | `TINYINT(1)` |
| `float` | `DECIMAL(p, s)`（金额）/ `FLOAT`（坐标） |
| `datetime` | `DATETIME` |
| `date` | `DATE` |
| `dict` / `list` | `JSON` |
| `bytes` | `BLOB` |

**金额**：用 `BIGINT` 存分（int），不用 float / decimal。

### 2.4 索引规范

| 索引类型 | 何时建 |
| --- | --- |
| 主键 | 必有 |
| 外键 | 必有 |
| `farm_id` | 业务表必有 |
| 唯一约束 | 业务唯一性（如 username） |
| 复合索引 | 高频查询 `(farm_id, created_at)` |
| 全文索引 | 中文搜索（暂未用） |

**禁止**：
- 在 `JSON` 字段建普通索引
- 单表索引超过 8 个
- 索引选择性 < 0.1（如 bool 字段）

### 2.5 JSON 字段

只在以下场景用 JSON：
- 扩展元数据（`metadata`）
- 半结构化数据（trace span、tool params）
- 不参与查询的字段

**禁止**：
- 把应该独立的字段塞 JSON
- 在 JSON 内做关联查询
- JSON 字段做 WHERE 过滤（用 generated column + 索引）

## 3. 迁移规范

### 3.1 工具

唯一允许的工具：**Alembic**。

```bash
# 生成迁移
poetry run alembic revision --autogenerate -m "add cost_categories table"

# 应用
poetry run alembic upgrade head

# 回滚
poetry run alembic downgrade -1

# 查看
poetry run alembic history
poetry run alembic current
```

### 3.2 迁移文件命名

```
backend/alembic/versions/
├── 2026_06_19_1000_add_cost_categories.py
├── 2026_06_19_1400_add_debt_table.py
└── ...
```

文件内：

```python
"""add cost_categories table

Revision ID: abc123
Revises: def456
Create Date: 2026-06-19 10:00:00
"""
```

### 3.3 迁移原则

| 原则 | 说明 |
| --- | --- |
| **小步前进** | 一个迁移只做一件事 |
| **可回滚** | 必须实现 downgrade |
| **不破坏数据** | 改字段类型先加新字段、迁数据、删旧字段 |
| **不锁表** | 大表 DDL 在低峰期；用 pt-online-schema-change |
| **测试覆盖** | 迁移跑在测试环境验证 |

### 3.4 禁止操作

- 手动 SQL 改表
- 直接修改 `models/*.py` 不生成迁移
- 跳过 Alembic 用 ORM `Base.metadata.create_all()`
- 在生产环境跑 autogenerate（必须在开发环境生成 + review）

## 4. 数据库连接

### 4.1 连接池

```python
# app/core/database.py
engine = create_engine(
    settings.DATABASE_URL,
    poolclass=QueuePool,
    pool_size=10,
    max_overflow=20,
    pool_timeout=30,
    pool_recycle=3600,
    pool_pre_ping=True,
)
```

### 4.2 Session

```python
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)

def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()
```

**禁止**：
- 全局共享一个 Session
- Session 跨请求
- 不关闭 Session（必须 finally）

## 5. ORM 规范

### 5.1 模型定义

```python
class Cost(Base):
    __tablename__ = "costs"

    id: Mapped[int] = mapped_column(primary_key=True)
    farm_id: Mapped[int] = mapped_column(ForeignKey("farms.id"), index=True)
    amount: Mapped[int] = mapped_column(BigInteger)  # 分
    category: Mapped[str] = mapped_column(String(50))
    record_date: Mapped[date] = mapped_column(Date)
    note: Mapped[str | None] = mapped_column(Text, nullable=True)
    created_at: Mapped[datetime] = mapped_column(default=datetime.utcnow)
    updated_at: Mapped[datetime] = mapped_column(
        default=datetime.utcnow,
        onupdate=datetime.utcnow
    )

    farm: Mapped["Farm"] = relationship(back_populates="costs")
```

### 5.2 查询规范

```python
# ✅ 带 farm_id
db.query(Cost).filter(
    Cost.farm_id == farm_id,
    Cost.record_date >= start_date,
).all()

# ✅ 用 join
db.query(Cost).join(Farm).filter(Farm.id == farm_id).all()

# ❌ 不带 farm_id（违反 Farm 隔离）
db.query(Cost).all()

# ❌ 字符串拼接 SQL
db.execute(f"SELECT * FROM costs WHERE farm_id = {farm_id}")
```

### 5.3 事务

```python
# Service 层管理事务
def create_cost(db: Session, schema: CostCreate, farm_id: int) -> Cost:
    cost = Cost(farm_id=farm_id, **schema.dict())
    db.add(cost)
    db.commit()
    db.refresh(cost)
    return cost
```

**禁止**：
- 在 API 层 commit
- 在 Skill 层 commit（让 Service 层做）
- 嵌套事务（用 savepoint 显式）

## 6. 数据保留与归档

| 数据 | 保留 |
| --- | --- |
| 业务核心（costs/cycles/...） | 永久 |
| 会话消息（conversation_messages） | 1 年 |
| Trace（trace_records） | 90 天热 + 1 年 JSONL |
| JSONL event log | 1 年 |
| 审计日志（audit_logs） | 1 年 |
| 待清理（pending_actions 过期） | 30 天 |

## 7. 备份

| 类型 | 频率 | 保留 |
| --- | --- | --- |
| MySQL 全量 | 每日凌晨 3 点 | 30 天 |
| MySQL binlog | 实时 | 7 天 |
| JSONL rsync | 每日凌晨 4 点 | 30 天 |

恢复演练：每季度一次。

## 8. 主备切换

生产环境（未来）：
- 主：读写
- 备：只读副本（异步复制）
- 故障切换：手动（避免脑裂）

当前 2C4G 单机：靠每日备份恢复。

## 9. 性能优化

| 场景 | 策略 |
| --- | --- |
| 慢查询 | 启用 slow_query_log，阈值 1s |
| 索引缺失 | EXPLAIN 分析，补索引 |
| N+1 查询 | 用 `selectinload` / `joinedload` |
| 大结果集 | 分页（必须带 farm_id） |
| 频繁读 | 应用层缓存（FarmContextCache） |

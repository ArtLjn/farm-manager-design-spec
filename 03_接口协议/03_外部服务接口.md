### 1.3 接入方式

```python
class LLMProvider(Protocol):
    async def chat(self, request: LLMRequest) -> LLMResponse: ...
    async def stream(self, request: LLMRequest) -> AsyncIterator[LLMChunk]: ...

class LLMClientManager:
    """根据 config 路由到具体 Provider，支持 fallback。"""
    providers: dict[str, LLMProvider]
    primary: str
    fallback: list[str]

    async def chat(self, request):
        try:
            return await self.providers[self.primary].chat(request)
        except (TimeoutError, ProviderError):
            for fb in self.fallback:
                try:
                    return await self.providers[fb].chat(request)
                except: continue
            raise AllProvidersFailedError
```

## 2. 天气服务

### 2.1 QWeather（主）

| 项 | 值 |
| --- | --- |
| 端点 | `https://devapi.qweather.com/v7` |
| 认证 | API Key（query 参数 `key=<QWEATHER_API_KEY>`） |
| 主要接口 | `/weather/now`、`/weather/7d`、`/weather/24h`、`/warning/now` |
| 城市定位 | 经纬度 → Location ID（`/geo/v2/city/lookup`） |
| 配额 | 免费 1000 次/天，付费 10 万次/天 |
| 缓存 | 5 分钟（current）、1 小时（forecast） |

### 2.2 Open-Meteo（兜底）

| 项 | 值 |
| --- | --- |
| 端点 | `https://api.open-meteo.com/v1` |
| 认证 | 无（免费） |
| 主要接口 | `forecast`（含 current + hourly + daily） |
| 优势 | 不需 API Key，全球覆盖 |
| 缺点 | 无农业建议、无中文描述 |

### 2.3 双源策略

```python
class WeatherService:
    async def get_current(self, lat: float, lon: float) -> WeatherSnapshot:
        try:
            return await self.qweather.get_current(lat, lon)
        except (TimeoutError, QuotaExceeded):
            return await self.open_meteo.get_current(lat, lon)
```

封装位置：`backend/app/services/weather_service.py`

### 2.4 农业天气预警

QWeather 提供 `/warning/now` 接口，可获取当地气象部门发布的预警（如霜冻、暴雨、高温）。

DataFlywheel 会基于历史预警 + 农户茬口生成「该不该浇水/打药/采收」建议。

## 4. MySQL（业务数据）

| 项 | 值 |
| --- | --- |
| 版本 | 8.x |
| 字符集 | utf8mb4 / utf8mb4_unicode_ci |
| 引擎 | InnoDB |
| 主备 | 生产环境主备（异步复制） |
| 连接串 | `mysql+pymysql://user:pass@host:3306/db?charset=utf8mb4` |
| 连接池 | SQLAlchemy QueuePool，size=10，max_overflow=20 |

**禁止**：
- 手动 SQL 改表（用 Alembic）
- 字符集非 utf8mb4
- 跨 farm_id 查询（必须带 farm_id WHERE）

## 6. systemd 服务

```ini
# /etc/systemd/system/<service_name>.service
[Unit]
Description=Farm Manager Backend
After=network.target mysql.service

[Service]
Type=simple
User=<deploy_user>
WorkingDirectory=<repo_dir>/backend
ExecStart=<repo_dir>/backend/.venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=3
Environment=PYTHONUNBUFFERED=1

[Install]
WantedBy=multi-user.target
```

具体路径、用户名、服务名不在设计 spec 中暴露，由部署仓库管理。

## 7. Nginx（反向代理）

```nginx
server {
    listen 443 ssl http2;
    server_name farm.example.com;

    ssl_certificate /etc/letsencrypt/live/farm.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/farm.example.com/privkey.pem;

    # Admin Web 静态资源
    location / {
        root /var/www/farm-admin;
        try_files $uri /index.html;
    }

    # API
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Trace-Id $http_x_trace_id;
    }

    # SSE 长连接
    location /api/v1/agent/chat/stream {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Connection "";
        proxy_buffering off;
        proxy_read_timeout 600s;
    }
}
```

## 2. 详细能力

### 2.1 自然语言记账

**用户故事**：作为农户，我希望一句话就能记账，不用打开表单选分类。

**输入示例**：
- 「上个月买了 200 块化肥，老王农资店赊账」
- 「昨天卖了 500 块番茄，现金」
- 「给老张开了 3000 块工资」

**输出**：
- Agent 解析为结构化数据并展示
- 写操作必须用户确认（Pending Action）
- 确认后写入 cost 表

**对应 Skill**：`create_cost_record`

**验收标准**：
- 金额、分类、日期、记录类型必须正确识别
- 赊账场景识别 counterparty + due_date
- 收入/支出分类正确
- 解析失败时清晰追问，不猜测

### 2.2 Smart Fill 表单

**用户故事**：作为农户，我希望复杂记账场景有结构化表单兜底，避免对话歧义。

**输入**：用户点击「记一笔」按钮

**流程**：
1. 调用 `GET /api/v1/scenarios` 拿场景列表（如「记账」「记工」「记事」）
2. 选择场景后渲染对应表单
3. 用户填表 → `POST /api/v1/parse` 提交
4. 后端调用 Agent application.smart_fill 处理

**对应代码**：`agent/application/smart_fill.py` + `api/smart_fill.py`

### 2.3 账单分析

**用户故事**：作为农户，我想知道这月花了多少、赚了多少、利润多少。

**输入示例**：
- 「这个月利润怎么样」
- 「今年化肥支出比去年多吗」
- 「最近 3 个月成本趋势」

**对应 Skill**：`get_cost_summary` / `get_cost_analytics`

**输出**：结构化数据 + 自然语言摘要

### 2.4 作物茬口管理

**用户故事**：作为农户，我想管理种了什么、什么阶段、什么时候收。

**输入示例**：
- 「种一茬番茄，1 号棚」
- 「当前茬口到什么阶段了」
- 「下茬推荐种什么」

**对应 Skill**：`create_crop_cycle` / `get_crop_cycles`

### 2.5 农事日志

**用户故事**：作为农户，我想记录每天干了什么农活。

**输入示例**：
- 「今天上午浇了水」
- 「昨天给 1 号棚打了杀菌药」
- 「这周做了哪些农事」

**对应 Skill**：`log_farm_activity` / `farm_logs`

### 2.6 派工与工资

**用户故事**：作为农户，我想管理工人和工作量，月底算工资。

**输入示例**：
- 「明天安排老张去 1 号棚浇水」
- 「老王本月工资多少」
- 「这个月开了多少工资」

**对应 Skill**：`create_operation_work_order` / `get_labor_payables`

### 2.7 天气决策

**用户故事**：作为农户，我想根据天气决定明天干什么。

**输入**：用户打开首页驾驶舱

**输出**：
- 当前天气（温度、湿度、风力、降水）
- 未来 3-7 天预报
- 基于天气 + 茬口 + 账单的农事建议

**对应代码**：`services/weather_service.py`（双源：QWeather + Open-Meteo）

### 2.8 AI 对话

**用户故事**：作为农户，我想跟 Yaya 助理对话，问问题、做决策。

**入口**：移动端 `features/yaya/`

**关键能力**：
- 多轮对话
- 工具调用（30+ Skill）
- 流式回复（SSE）
- Pending 确认（写操作）
- 人设切换（professional / warm / creative）

### 2.9 每日建议

**用户故事**：作为农户，我打开 App 就能看到今天该干什么。

**触发**：
- 每日定时任务（凌晨）
- 用户主动刷新

**输入**：当前茬口 + 天气 + 最近账单 + 农事日志

**输出**：3-5 条结构化建议（如「明天可能下雨，今天安排打药」「番茄进入膨果期，建议追肥」）

**对应代码**：`agent/application/daily_advice_use_case.py`

### 2.10 周期报告

**用户故事**：作为农户，我想周末/月末收到一份报告，知道这周/月情况。

**对应代码**：`agent/application/report_use_case.py`

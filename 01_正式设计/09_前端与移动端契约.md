## 1. 三端分工

| 端 | 用户 | 主用途 | 技术 |
| --- | --- | --- | --- |
| **Flutter App** (`mobile-app/`) | 农户 | 日常记账、查茬口、看建议、对话 | Flutter + Dart 3 |
| **Admin Web** (`admin-web/`) | 团队内部 | Skill 管理、Trace 调试、DataFlywheel、Token 监控 | React 19 + TS + Vite + Ant Design 5 |
| **Playground**（Admin Web 内） | 团队 + 内测 | 复现用户输入、调试 Agent | React（同 Admin） |

注意：README 标注移动端 React Native 已过期，实际为 Flutter。

## 2. 共享 API 契约

三端共用同一套 REST API：
- 路径前缀：`/api/v1/`
- 认证：JWT in `Authorization: Bearer <token>`
- 错误格式：`{code: str, message: str, detail?: object}`
- 分页：`?page=1&page_size=20`，返回 `{items, total, page, page_size}`
- 时间：ISO 8601 UTC，前端转本地时区
- 货币：分（int），前端展示除以 100；金额输入校验后端做

详见 [03_接口协议/01_HTTP_API协议](../03_接口协议/01_HTTP_API协议.md)。

### 3.2 设计系统

权威：[.claude/rules/app-ui.md](../../.claude/rules/app-ui.md)

核心：
- **极简主义**：留白、清晰、呼吸感
- **一个焦点**：每屏一个主视觉焦点
- **信息密度低**：短标签、轻描述
- **8pt 栅格**：间距 4/8/12/16/24/32/40
- **20px 屏幕边距**，16-20px 卡片内边距
- **24px+ 段间距**
- **单层卡片**，避免嵌套
- **圆角 20-28px**
- **阴影**：`0 4px 12px rgba(0,0,0,0.04)`
- **字体**：SF Pro / Inter，4 种字号、2 种字重
- **颜色**：主色 `#4F8CFF`，背景 `#F7F8FA`，表面 `#FFFFFF`，60/30/10 用色比
- **图标**：单一风格（Lucide 或 Phosphor），outline 或 filled 不混用
- **AI 模块轻量**：不巨大机器人头标，不堆 AI 标语

### 3.3 状态管理

| 场景 | 方案 |
| --- | --- |
| 简单局部状态 | `StatefulWidget` + `setState` |
| 跨页面共享 | Riverpod / Provider |
| 服务端状态 | Repository + Future/Stream |
| 表单状态 | 自管理 + Controller |

### 3.4 路由

`go_router`（推荐）或 `Navigator 2.0`：
- 深链接支持
- 嵌套导航（BottomNavigationBar）
- 路由守卫（登录态、农场态）

### 3.5 网络层

```dart
// data/api/api_client.dart
class ApiClient {
  Future<T> get<T>(String path, {Map<String, dynamic>? query});
  Future<T> post<T>(String path, {required Map<String, dynamic> body});
  Stream<SseEvent> stream(String path, {required Map<String, dynamic> body});
}
```

约束：
- 全局错误拦截（401 跳登录）
- trace_id 透传（header: `X-Trace-Id`）
- 重试策略（5xx 重试 1 次）
- 超时：connect 5s，receive 30s，SSE 不超时

### 4.3 编码规范

权威：[.claude/rules/frontend-style.md](../../.claude/rules/frontend-style.md)

核心：
- 函数组件 + Hooks
- Props 用 interface 定义，禁止 any
- async/await，禁止 .then 链
- 组件超 150 行考虑拆分
- 文件名 PascalCase（组件）/ camelCase（工具）
- vitest + React Testing Library 测试
- 重点测交互，不测样式

## 5. 三端一致性原则

| 维度 | 原则 |
| --- | --- |
| 业务概念 | 三端用同一术语（如「茬口」不写成「种植周期」） |
| 字段单位 | 三端用同一单位（如金额分） |
| 错误信息 | 后端返回的中文 message 直接展示，前端不二次翻译 |
| 枚举 | 后端为单一来源，前端 enum 与之同步 |
| 时间 | 后端 UTC ISO，前端转本地 |
| 空状态 | 三端统一文案（如「暂无账单」） |

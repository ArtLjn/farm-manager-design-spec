# 02 — 回归测试集

> 状态：草稿 | 维护：BlockShip | 关联：[01_正式设计/06_数据飞轮与评测](../01_正式设计/06_数据飞轮与评测.md)、权威 [../../docs/architecture/data-flywheel-repair-pack-workflow.md](../../docs/architecture/data-flywheel-repair-pack-workflow.md)

---

## 1. 目的

每次代码合并到 `main` 前，自动验证 Agent 关键行为不回归。

## 2. 用例来源

| 来源 | 占比 | 说明 |
| --- | --- | --- |
| DataFlywheel 高置信样本 | 60% | 经人工确认的真实用户轨迹 |
| 历史 bug 复现 | 20% | 每个 prod bug 必须留下回归用例 |
| 边界 / 异常 | 15% | 缺参、越权、超长输入、并发 |
| 新功能 | 5% | 上线后转回归 |

## 3. 用例结构

位于 `backend/simulation/cases/<scenario_id>.json`：

```json
{
  "scenario_id": "cost-create-with-debt",
  "description": "用户记账并指定赊账对象",
  "user_input": "昨天买了200块化肥，老王农资店赊账",
  "context": {
    "farm_id": 1,
    "current_crop_cycle_id": 12
  },
  "expected": {
    "skills_called": ["create_cost_record"],
    "pending_action": {
      "type": "create_cost",
      "payload_contains": {"amount": 200, "category": "化肥", "counterparty": "老王农资店"}
    },
    "response_contains": ["已记账", "赊账", "老王"],
    "reflector_warnings": []
  },
  "tags": ["cost", "debt", "pending_action"]
}
```

## 5. 运行方式

### 5.1 命令

```bash
# 跑全集
poetry run python -m app.simulation.run --suite regression

# 跑 smoke（< 60s）
poetry run python -m app.simulation.run --suite smoke

# 跑指定 tag
poetry run python -m app.simulation.run --tags cost,debt

# 单条用例
poetry run python -m app.simulation.run --case cost-create-with-debt
```

### 5.2 输出

```text
[regression] 87/90 passed (96.7%) — 3 failed

FAILED scenarios:
  - cost-create-with-debt: missing "赊账" in response
  - cycle-current-status: wrong skill called (expected get_cycle_status, got get_cost_summary)
  - edge-future-date: should return NEED_CLARIFY but got SUCCESS

Report: backend/simulation/reports/2026-06-19_153020.html
```

报告包含：
- 用例明细 + 实际 vs 预期 diff
- 调用的 Skills / Tools / LLM 详情
- Token 消耗
- Reflector 警告
- Trace 链接

## 6. 回归判定

| 字段 | 判定方式 |
| --- | --- |
| `skills_called` | 集合相等（顺序无关） |
| `pending_action.type` | 严格相等 |
| `pending_action.payload_contains` | 子集匹配 |
| `response_contains` | 每个关键词都出现 |
| `reflector_warnings` | 必须为空 |

**任一字段不匹配 → 用例失败。**

## 7. 失败处理

1. CI 失败 → 阻塞合并。
2. 开发者查看报告，定位根因。
3. 修复后本地复跑。
4. 若用例本身过时（如 Prompt 文案变更），更新用例 + 添加 changelog 注释。
5. 不允许通过删除用例让 CI 变绿。

## 8. 用例维护节奏

| 频率 | 动作 |
| --- | --- |
| 每次 Prompt 大改 | 全量回归 |
| 每周 | 跑 full 回归 + 评测 |
| 每月 | 从 DataFlywheel 选 5-10 个高价值样本进入回归集 |
| 每季度 | 审视并淘汰低价值用例（保留至少 50 条核心） |

## 9. 与 DataFlywheel 的关系

详见 [01_正式设计/06_数据飞轮与评测](../01_正式设计/06_数据飞轮与评测.md)。

- DataFlywheel 样本 → 人工确认 → 标记 `regression_candidate: true` → 进入候选池。
- 候选池样本由 PM/工程师评审 → 入回归集。
- 入集后该样本从飞轮训练池移除（避免泄漏）。
